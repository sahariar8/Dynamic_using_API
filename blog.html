Question: Discuss the scope of var, let, and const?

Ans:
var, let, and const are three different ways to declare variables in JavaScript, and they have different scopes and behaviors:
var:
1. Function-scoped: Variables declared with var are function-scoped, meaning they are only accessible within the function in which they are declared.
2. Hoisting: Variables declared with var are hoisted to the top of their containing function or global scope, which means they are moved to the top during the compilation phase. This can lead to unexpected behavior.
3. Can be redeclared: Variables declared with var can be redeclared within the same scope without any errors.
4. No block scope: var does not have block-level scope, which means it is accessible outside of blocks.
javascript.

let:
1. Block-scoped: Variables declared with let have block-level scope, meaning they are only accessible within the block in which they are defined.
2. Hoisting: Like var, variables declared with let are hoisted to the top of their containing block, but they are not initialized until they are encountered in the code. Accessing a let variable before its declaration will result in a ReferenceError.
3. Cannot be redeclared: Variables declared with let cannot be redeclared in the same scope.

const:
1. Block-scoped: Like let, variables declared with const are block-scoped.
2. Hoisting: Similar to let, const variables are hoisted to the top of their containing block but are not initialized until they are encountered in the code. Accessing a const variable before its declaration will result in a ReferenceError.
3. Cannot be reassigned: Variables declared with const cannot be reassigned after their initial assignment. They are constants and maintain their value throughout their lifetime.

Question:Tell us the use cases of null and undefined?

Ans:
null and undefined are two distinct JavaScript values that are often used to represent missing or empty data, but they have different use cases:
null:
1. null is a value that represents the intentional absence of any object value or a variable that has been explicitly set to indicate no value or no reference.
2. It is typically used when you want to explicitly signal that a variable or property does not currently hold a value.
3. It is a valid value to assign to a variable to indicate that it is intentionally empty or uninitialized.
4. null is often used in situations where you want to clear out a variable or object property, indicating that it should not reference any object or have any value.

undefined:
1. undefined is a value that is automatically assigned to a variable or property if you try to access or query a variable or property that has not been initialized or does not exist.
2. It is used to indicate that a variable has been declared but has not been assigned any value.
3. It can also be the default return value of a function that does not explicitly return anything.
4. It's important to note that undefined is a value that JavaScript assigns, and it is distinct from null.

Question: What do you mean by REST API?

It is not a technology itself but rather a style or approach for building web services that are scalable, stateless, and easy to maintain. Key characteristics and principles of RESTful APIs include:

1.Statelessness: In a REST API, each request from a client to the server must contain all the information needed to understand and process the request. The server should not rely on any previous requests or session state stored on the server. This makes RESTful services easy to scale and maintain.

4.HTTP Status Codes: REST APIs use HTTP status codes to indicate the outcome of a request. For example, a 200 status code typically indicates success, while a 404 status code indicates that the requested resource was not found.
